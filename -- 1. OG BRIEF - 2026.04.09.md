Project Brief

FIND AI / “There is no ‘AI’ in DESIGN”

1. Project summary

This project is a small, clever, interactive website built as a search-based experience with a conceptual reveal.

At first glance, it behaves like a game. In reality, it is a designed manifesto.

The user enters the site, sees a bold instruction to Find AI, and is presented with a giant inspectable DESIGN graphic placed on a large canvas. They are asked to search for a specific symbol hidden somewhere inside that graphic within a short time limit. They can pan and zoom across the canvas and click where they believe the symbol is.

The catch is that the symbol is not there.

When time ends, the site reveals the message:

There is no “AI” in DESIGN

It then expands into the full manifesto text.

The experience is meant to be:
	•	conceptually sharp
	•	visually memorable
	•	easy to understand
	•	simple to interact with
	•	more like an interactive statement piece than a casual game

This project should feel like a provocation with craft, not like a joke, toy, or gimmick.

⸻

2. Core idea

The project is built on a simple tension:

People increasingly talk about AI as if it is the center of design.

This experience invites the user to literally search for “AI” inside DESIGN.

They search.
They zoom.
They inspect.
They spend effort trying to find it.

Then the experience reveals the thesis:

There is no “AI” in DESIGN because design was never about the tool.

This does not mean AI does not matter.
It does not mean AI should not be used.
It does not mean design workflows will remain unchanged.

It means that:
	•	tools are not the essence of design
	•	output is not the same as design
	•	design begins where judgment, intent, taste, clarity, and responsibility begin

The project should communicate this through interaction first, then through text.

⸻

3. Strategic goal

The website should function as:
	•	a self-contained interactive manifesto
	•	a memorable digital artifact
	•	a point of view made tangible
	•	a portfolio-grade piece that reflects strong creative direction and product thinking

It should not feel like an educational microsite.
It should not feel like a startup landing page.
It should not feel like an anti-AI rant.
It should not feel like a “Where’s Wally” clone, even though the interaction logic is inspired by that kind of search mechanic.

The goal is to create an experience that makes users perform the misunderstanding before revealing the message.

⸻

4. Core thesis

This is the intellectual backbone of the project:
	•	AI changes tools and workflows
	•	AI increases speed, access, and output
	•	but design is not reducible to output
	•	design is fundamentally about judgment
	•	when generation becomes easier, fundamentals matter more
	•	taste, clarity, experience, and decision-making become more important, not less

The site must preserve this nuance.

The thesis is not:
	•	“AI is fake”
	•	“AI has no place in design”
	•	“real designers never use AI”
	•	“technology is the enemy”

The thesis is:
	•	AI is a tool
	•	design is not the tool
	•	the tool does not replace judgment

That distinction is essential.

6. Desired emotional arc

The experience should produce a specific sequence of feelings:

1. Curiosity

The user lands and immediately understands there is a challenge.

2. Focus

The user starts searching and becomes temporarily absorbed in the task.

3. Mild tension

The countdown creates urgency. The user believes there is something to find.

4. Frustration or uncertainty

They cannot locate the symbol.

5. Realization

The site reveals that the symbol was never there.

6. Reframing

The user understands the statement is about the nature of design, not about winning or losing a game.

7. Reflection

The manifesto lands as the explanation of what the interaction meant.

This emotional sequence matters more than visual spectacle alone.

⸻

7. Experience structure

The website is a single linear experience with clear states.

State A — Landing / Idle

When the user first enters the site:
	•	they see a bold central headline "Find AI" where "AI" will most likely will not be text but graphic / svg showing exact element / symbol users are suppose to look for
	•	below it, they see the large DESIGN graphic or a preview of it on the canvas
	•	the giant DESIGN world is visible, but not yet controllable
	•	the interface already contains the four corner UI elements
	•	there is a clear CTA to begin the experience

The initial state should feel clean, minimal, and slightly mysterious.

Important:
	•	the reveal headline must not be visible here
	•	the user must not feel overloaded with explanation
	•	the premise should be immediately understandable

The opening state should communicate:
	•	this is a focused interactive challenge
	•	the user is being asked to search for something
	•	the central object is the large DESIGN composition

⸻

State B — Start / Challenge activation

Once the user starts:
	•	the timed challenge begins
	•	the canvas becomes interactive
	•	the user can move around and inspect the DESIGN composition
	•	the target symbol becomes relevant as the object they are supposed to find

This moment should feel crisp and intentional.

There can be a very short instruction layer, but it should not become verbose or slow.

The user should quickly understand:
	•	I have a short amount of time
	•	I can pan and zoom
	•	I need to click where I think the target is

⸻

State C — Search phase / Timed interaction

This is the active part of the experience.

The user is given:
	•	a limited time window, default 30 seconds
	•	the ability to navigate the large canvas
	•	the ability to zoom into the DESIGN world
	•	the ability to click where they think the hidden symbol is

The core behavior here is searching.

This phase should feel:
	•	frictionless
	•	intuitive
	•	focused
	•	absorbing
	•	visually rich

The user should not be distracted by unnecessary UI or nonessential features.

The point of this phase is not difficulty for its own sake.
The point is to create real engagement with the artwork and with the premise.

The user should believe, at least for a moment, that the target exists.

⸻

State D — Timeout / End of search

When time runs out:
	•	the interaction ends
	•	the user can no longer continue searching
	•	the experience transitions clearly into the reveal phase

The transition should feel deliberate.
Not abrupt in a broken way, and not over-dramatic either.

This should mark a shift from:
	•	searching
to
	•	understanding

⸻

State E — Reveal & manifesto

Once the search phase ends, the site reveals "There is no “AI” in DESIGN because design was never about the tool." headline and manifesto below it

This is the core conceptual turn.

The user must understand that:
	•	they did not fail because they were careless
	•	they did not miss the symbol
	•	the symbol was never there

The reveal should frame the experience as a statement, not as a prank.

This is a crucial difference.

The site should not say:
	•	gotcha
	•	surprise
	•	you lost

It should say, in effect:
	•	you were invited to search for the wrong thing
	•	that is the point

This is where the project becomes meaningful.

Manifesto:

The manifesto should feel like:
	•	the explanation
	•	the payoff
	•	the articulation of what the interaction meant

It should not feel like a random block of text added after the fact.

The site should now move from interactive provocation to clear statement.

The manifesto text is final and should be treated as approved source copy.

⸻

8. Approved manifesto text below "There is no “AI” in DESIGN because design was never about the tool." headline

Use this exact copy:

"Tools change. Fast. New ones arrive every week. AI will change how design is made. It will speed up output, expand access, and raise the floor.

But output is not the same as design. Design is deciding what matters. What problem is worth solving. What outcome is worth pursuing. What to simplify. What to keep. What to make clear.

Better tools make fundamentals matter more, because when everyone can generate, judgment becomes the edge. Taste becomes the filter. Experience becomes leverage. And design is what turns capability into advantage.

So use AI. Push it. Learn it. Just do not confuse operating tools with knowing what good looks like."

Do not rewrite this text unless explicitly instructed later.

9. The DESIGN world / artwork

This is the heart of the project.

The central graphic is a giant composition built around the word:

DESIGN

The word itself must become a large inspectable world on the canvas.

This world should contain:
	•	hundreds or thousands of small details
	•	visual micro-elements
	•	layered environments
	•	dense but intentional information
	•	enough detail to reward zooming and searching

The graphic should be authored and designed later by Dawid, but Codex must understand what role it plays in the experience.

This is not decoration.
This is not filler.
This is not a background.

It is the central artifact of the site.

The graphic must satisfy three levels at once:

Macro level

From a distance, it must clearly read as the word DESIGN.

Mid level

As the user zooms in, the letters should reveal internal structure, variation, visual logic, districts, or zones.

Micro level

At close range, the user should discover small, dense, interesting visual details worth inspecting.

The graphic must feel intentionally composed.
It cannot feel generic, random, or like uncurated AI clutter.

Even if AI tools are used in the production pipeline, the final visual should feel:
	•	art directed
	•	selective
	•	authored
	•	coherent
	•	meaningful

The project becomes weaker if the artwork feels like “detail for detail’s sake.”

The world should evoke the idea that DESIGN contains:
	•	systems
	•	choices
	•	thinking
	•	process
	•	refinement
	•	labor
	•	clarity
	•	iteration
	•	tension
	•	craft

The target symbol must not be present anywhere inside this world.

That absence is the whole point.

10. Target symbol

The object the user is told to search for should be a specific symbol or mark associated with the idea of “AI”.

It should:
	•	be visually clear
	•	be easy to recognize quickly
	•	be strong enough to search for mentally
	•	not resemble a specific third-party company brand too closely
	•	not create legal or aesthetic baggage
	•	feel native to the project

It can be minimal, icon-like, symbolic, or slightly custom.

Its job is simple:
	•	it must be believable as a hidden-object target
	•	it must be memorable enough for the user to search for it
	•	it must be fully absent from the actual composition

The target should not be too ornate or ambiguous.
If it is too complex, the user will not know what they are looking for.
If it is too generic, the concept becomes weak.

11. Interaction philosophy

The site should be simple to use without explanation.

The interaction model is essentially:
	•	start
	•	inspect
	•	navigate
	•	guess
	•	reveal
	•	read

The user should be able to:
	•	pan the canvas
	•	zoom in and out
	•	inspect details
	•	click where they think the symbol is

The interaction should feel:
	•	direct
	•	smooth
	•	precise
	•	premium
	•	controlled

It should not feel:
	•	gamey in a cheap way
	•	overloaded with mechanics
	•	overengineered
	•	chaotic

The canvas can feel large and expansive.

The user should experience it as:
a big inspectable world.

That is enough.

12. Time limit

Default time limit:
30 seconds

This is the correct default for the concept.

Why:
	•	long enough to create engagement
	•	short enough to avoid irritation
	•	creates urgency without becoming punishment
	•	helps the reveal land sharply

The project should not default to 1 or 2 minutes.
That would dilute the concept and make the search feel tedious.

This experience should be tight.

15. Important conceptual guardrails

These guardrails matter. Codex should understand them clearly.

1. This is not anti-AI

The site is not arguing against using AI.
The manifesto explicitly says:

So use AI. Push it. Learn it.

That line is essential.
It protects the work from reading as fear-based or outdated.

2. This is not hypocrisy to hide

It is acceptable, even useful, that AI and non-design tools may help create parts of the site.

That paradox strengthens the concept:
the project can be built with AI, but still argue that AI is not the source of design judgment.

That is a feature, not a problem.

3. The user should not feel mocked

The reveal should reframe, not humiliate.
The point is not “you were stupid.”
The point is “you were invited into a false assumption, and now you see why.”

4. The artwork must carry meaning

If the DESIGN world feels empty, arbitrary, or generic, the site weakens dramatically.
The entire argument depends on the world feeling rich, authored, and worthy of inspection.

5. This is not a novelty mini-game

The interaction is a vehicle for meaning.
It is not the end in itself.

18. What success looks like

The project succeeds if a first-time user:
	•	immediately understands there is a search challenge
	•	becomes engaged enough to inspect the artwork
	•	feels genuine tension during the countdown
	•	reaches the reveal and understands its meaning
	•	sees the manifesto as the natural payoff
	•	remembers the idea after leaving the site

The project especially succeeds if the user walks away thinking something like:

“Right. AI may be part of the workflow, but it is not the thing that makes design design.”

That is the right takeaway.

⸻

19. What failure looks like

The project fails if users interpret it as:
	•	a shallow anti-AI slogan
	•	a generic hidden-object website
	•	an overdesigned gimmick
	•	a joke that wastes 30 seconds
	•	a hypocritical statement because AI tools were used in production
	•	a concept with weak or lazy visual execution

The site must avoid those readings through:
	•	clarity
	•	tone
	•	restraint
	•	a strong reveal
	•	meaningful artwork
	•	manifesto nuance

20. Scope discipline

This project should stay small and sharp.

That means:
	•	one core interaction
	•	one central artwork
	•	one short timed search
	•	one reveal
	•	one manifesto payoff

It does not need:
	•	extra game levels
	•	leaderboards
	•	rewards
	•	hidden achievements
	•	branching narratives
	•	excessive transitions
	•	feature creep

The power of the project comes from precision, not volume.
